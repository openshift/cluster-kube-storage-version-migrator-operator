# Architecture: cluster-kube-storage-version-migrator-operator

## Scope

This operator manages the **kube-storage-version-migrator** operand in OpenShift. The operand's job is to re-write Kubernetes objects stored in etcd so they use the current storage version after API server upgrades. The operator itself does not perform migrations — it ensures the migrator Deployment runs correctly and reports health via ClusterOperator status.

The trigger controller (which automatically creates StorageVersionMigration requests) is intentionally **not deployed**. Migration requests are created by resource owners using `StorageVersionMigration` CRs (`migration.k8s.io/v1alpha1`).

## Namespace Map

| Namespace | Contents |
|-----------|----------|
| `openshift-kube-storage-version-migrator-operator` | Operator Deployment, ServiceAccount, ConfigMap, NetworkPolicies |
| `openshift-kube-storage-version-migrator` | Operand (migrator) Deployment, ServiceAccount, RBAC, PDB, NetworkPolicies |

## Manifest vs Bindata Split

```text
manifests/          → Installed by CVO into the operator namespace
                      (CRDs, operator Deployment, SA, RBAC, ClusterOperator, NetworkPolicies)

bindata/            → Applied by the operator at runtime into the operand namespace
                      (namespace, SA, RBAC, PDB, NetworkPolicies, migrator Deployment)
```

CVO owns the operator lifecycle. The operator owns the operand lifecycle.

## Controllers

`RunOperator()` in `pkg/operator/starter.go` starts six controllers:

```text
RunOperator(ctx, controllerContext)
  ├── StaticResourceController          (library-go)
  │     Applies bindata: namespace, SA, RBAC, PDB, NetworkPolicies
  │
  ├── MigratorDeploymentController      (local + library-go)
  │     Reconciles operand Deployment with hooks:
  │       • Image injection (${IMAGE} → env IMAGE)
  │       • Log level propagation (operator spec → --v=N)
  │       • Control-plane node selector (standard topology)
  │       • Replica count (= control-plane nodes, min 1; HyperShift → 2)
  │
  ├── ClusterOperatorStatusController   (library-go)
  │     Reports Available/Progressing/Degraded on the ClusterOperator
  │
  ├── StaticConditionsController         (local)
  │     Sets DefaultUpgradeable=True unconditionally
  │
  ├── RemoveStaleConditionsController   (library-go)
  │     Prunes conditions no longer reported by any controller
  │
  └── LoggingController                 (library-go)
        Propagates LogLevel from operator spec to klog verbosity
```

## Reconciliation Flow

```text
Event (KubeStorageVersionMigrator CR, Deployment, Secret, Node, Infrastructure)
  → Informer enqueues work item
    → Controller sync()
      ├── StaticResourceController: ensure bindata resources match desired state
      ├── DeploymentController:
      │     1. Load deployment manifest from bindata
      │     2. Replace ${IMAGE} with operand image from env
      │     3. Set --v=N based on operator LogLevel
      │     4. Set nodeSelector (control-plane, unless HyperShift)
      │     5. Set replicas (control-plane count, or 2 for HyperShift)
      │     6. Apply deployment, report generation status
      └── StatusController: aggregate conditions → ClusterOperator
```

|-----|-----------|--------|
| `StorageVersionMigration` | `migration.k8s.io` | Vendored from `sigs.k8s.io/kube-storage-version-migrator` (OpenShift fork) |

CRD YAMLs in `manifests/` are verified against vendored copies via `make verify`.

## HyperShift / Topology Behavior

The deployment controller checks `Infrastructure` resource topology:

- **Standard (ControlPlaneTopology=HighlyAvailable):** Pins operand to control-plane nodes, replicas = node count.
- **External (HyperShift):** No node selector, replicas = 2, control-plane scheduling disabled.

E2E tests are Ginkgo-based, registered via OTE, and require a live OpenShift cluster. Tests are tagged:
- No tag → parallel suite
- `[Serial]` → serial suite (parallelism=1)
- `[Disruptive]` → disruptive suite (parallelism=1, ClusterStabilityDisruptive)

## Design Decisions

1. **No trigger controller** — Prevents uncontrolled mass migrations; owners opt in via explicit CRs.
2. **cluster-admin for operand** — Migrator must read/write all resource types across all namespaces.
3. **library-go over custom logic** — Maximizes consistency with other OpenShift operators; minimizes maintenance.
4. **Static conditions** — `DefaultUpgradeable=True` avoids blocking cluster upgrades.
5. **Network policies** — Default-deny + explicit allow for egress (migrator needs API server access) and metrics ingress (operator needs scraping).
